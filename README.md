# QHadoop Tools
QHadoop Tools is a QGIS plugin that enables interaction with Hadoop (HDFS) through WebHDFS, designed to support spatial data workflows such as those generated by the Spatial Framework for Hadoop (Esri).

---

## 🚀 Features

- Download files from HDFS to local environment
- Upload files from local environment to HDFS
- Real-time progress tracking for upload and download operations
- Automatic handling of HDFS directory outputs (e.g., `part-*`)
- Fix and standardize JSON / GeoJSON files
- Input validation for connection parameters and paths
- Automatic retry mechanism for network operations

---

## 🌍 Context

This plugin was originally designed to support spatial data workflows based on the **Spatial Framework for Hadoop (SFH)** developed by Esri for large-scale geospatial analytics.

In distributed processing environments such as Hadoop, Spark, or Hive, spatial data is typically generated as:

- Partitioned outputs (e.g., `part-*` files)
- JSON-based structures not directly compatible with GIS tools
- Geometry formats that require transformation before visualization

These outputs are not immediately usable in desktop GIS applications like QGIS.

QHadoop Tools was created to bridge this gap by:

- Enabling direct interaction with HDFS through WebHDFS
- Downloading and consolidating distributed outputs into local files
- Transforming JSON outputs into valid GeoJSON
- Providing tools to fix geometry structures for GIS compatibility

This allows users to efficiently move from large-scale distributed spatial processing to visualization and analysis within QGIS.

The plugin therefore serves as a practical integration layer between **big data spatial processing frameworks and traditional GIS workflows**.

This workflow is common in big data geospatial pipelines, where results must be transferred from distributed systems into desktop GIS tools for analysis.

---

## 🏗 Architecture

The plugin follows a layered architecture:

**UI (Dialogs / Views)** → **Controller** → **Tasks (QgsTask)** → **Services** → **Client** → **WebHDFS**
 
---

## ⚙️ Requirements

- QGIS 3.x  
- Hadoop cluster with WebHDFS enabled  
- Network access to NameNode  

---

## 🔌 Configuration

- **Host** → NameNode hostname or IP  
- **Port** → 50070 (Hadoop 2.x) or 9870 (Hadoop 3.x)  
- **User** → HDFS user  

---

## ▶️ Usage

1. Open **QHadoop Tools** in QGIS  
2. Configure connection parameters  
3. Select an operation:
   - Upload to HDFS  
   - Download from HDFS  
   - Fix GeoJSON  
4. Execute the task  

✅ Runs in background  
✅ Progress visible in QGIS Task Manager  

---

## ⚠️ Notes

- Large file transfers may require increased timeout depending on network conditions
- Ensure WebHDFS is enabled (`dfs.webhdfs.enabled=true`)
- Output files are saved in `.json` format
- Poor network or HDFS overload may cause connection reset errors

## 📜 Evolution & History

### v0.1 (Initial implementation)

- Initial implementation built using QGIS Plugin Builder
- Monolithic structure combining UI, business logic, and HDFS communication
- Implemented core operations:
  - Upload files to HDFS
  - Download files from HDFS
  - Basic JSON / GeoJSON adjustment
- Initial integration with WebHDFS
- Served as a proof of concept for spatial data workflows in Hadoop environments

---

### v1.0 (Initial Refactor)

- Major architectural refactor from a monolithic prototype to a layered design
- Separation of concerns across distinct components:
  - UI (dialogs and views)
  - Controller (orchestration layer)
  - Services (business logic)
  - Infrastructure (WebHDFS communication)
- Improved code organization, maintainability, and scalability
- Established a solid foundation for future enhancements such as progress tracking

---

### v1.1

- Refactor of the UI layer to improve structure and maintainability
- Separation of interface components into:
  - `views/` → Qt Designer files (`.ui`)
  - `dialogs/` → UI logic, validation, and interaction with the controller
- Improved organization of the user interface layer
- Introduced an issue related to incorrect UI file path resolution after restructuring

---

### v1.2

- Fix for UI dialog loading issues introduced in v1.1
- Corrected `.ui` file path resolution after the separation between `views/` and `dialogs/`
- Stabilization of the UI layer after refactor
- Ensured proper loading of dialogs within QGIS environment

---

### v2.0 (Stability & UX Improvements)

- Introduction of real-time progress tracking for both upload and download operations
- Integration with QGIS background task system using `QgsTask`
- Improved user experience by preventing UI blocking during long operations
- Enhanced stability in WebHDFS communication, including better handling of redirects and streaming
- Implementation of a retry mechanism with exponential backoff for network operations
- Improved handling of distributed outputs from Hadoop (e.g., `part-*` files)
- Integration of a centralized logging system across all layers
- Consolidation of a layered architecture established in previous versions
- Increased reliability for large file transfers between local environment and HDFS

---
